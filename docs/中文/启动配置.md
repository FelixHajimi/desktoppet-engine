## 启动配置

启动配置存储在根目录的 `setting.json` 文件中。该文件控制程序启动时的全局行为。

```json
{
  "desktopPet": "Example",
  "dataDir": "data",
  "imageSize": [128, 128],
  "language": "zh-cn",
  "logPath": "./last.log",
  "logLevel": 0
}
```

---

### `desktopPet`

启动时需要加载的桌宠目录名称。

程序会根据该名称在 `dataDir` 指定的目录下查找对应的桌宠文件夹。例如 `"Example"` 表示加载 `data/Example/` 目录下的桌宠。

---

### `dataDir`

桌宠数据目录名称。

程序会在该目录下查找所有桌宠包文件夹。默认为 `data`。

---

### `imageSize`

桌宠动画尺寸，格式为 `[宽度, 高度]`。

程序会自动将动画缩放到该尺寸显示。默认为 `[128, 128]`。

---

### `language`

程序使用的界面语言。影响日志输出、右键菜单、对话框以及插件界面。

目前支持的语言：

| 语言             | 代码    |
| ---------------- | ------- |
| 中文（中国大陆） | `zh-cn` |
| 中文（台湾地区） | `zh-tw` |
| 英语（美国）     | `en-us` |
| 日语（日本）     | `ja-jp` |
| 韩语（韩国）     | `ko-kr` |
| 俄语（俄罗斯）   | `ru-ru` |
| 法语（法国）     | `fr-fr` |

---

### `logPath`

日志文件路径。

程序运行时的所有日志输出将写入该文件。默认为 `./last.log`。

---

### `logLevel`

日志输出等级，控制哪些级别的日志被记录。

| 值  | 级别     | 输出内容                 |
| --- | -------- | ------------------------ |
| 0   | debug    | 输出所有日志             |
| 1   | info     | 输出 info 及以上         |
| 2   | warn     | 输出 warning 及以上      |
| 3   | error    | 只输出 error 和 critical |
| 4   | critical | 只输出 critical          |

默认为 `0`，即输出所有日志。调高此值可屏蔽低级别日志，减少日志文件体积。
