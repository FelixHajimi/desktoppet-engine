## 常见问题

---

**Q: 日志文件在哪里？**

**A:** 默认在程序根目录下的 `last.log`。可在 `setting.json` 中通过 `logPath` 自定义路径。

---

**Q: 弹窗提示"桌宠启动过程中出现错误，具体信息查看日志"，是什么情况？**

**A:** 这是桌宠在启动过程中捕获到异常后弹出的错误提示，具体原因需要查看 `last.log` 日志文件中的详细错误信息。常见原因包括配置文件格式错误、资源文件缺失、插件加载失败等。如果看不懂日志内容，可以将日志文件发给开发者协助排查。

---

**Q: 为什么我DIY的languageMap文件不生效？**

**A:** 检查以下几点：

1. 语言代码是否匹配 —— `setting.json` 中的 `language` 字段值必须与 `languageMap.json` 中的语言键名完全一致（如 `"zh-cn"`、`"en-us"`）
2. JSON 格式是否正确 —— 缺少逗号、多出逗号、引号不匹配都会导致解析失败
3. 翻译键名是否正确 —— 程序通过键名（如 `"program.menu.exit"`）查找对应的翻译文本，键名必须与代码中调用的完全一致
4. 是否重启了程序 —— `languageMap.json` 在程序启动时加载，修改后需要重启才能生效
5. 日志中是否有翻译相关的错误 —— 查看 `last.log` 中是否有 `TRANSLATE ERROR` 的提示

如果以上都检查无误，可以把你的 `languageMap.json` 内容发给开发者看看。

---

**Q: 怎么玩别人的桌宠包？**

**A:** 将别人的桌宠包文件夹放到 `data/` 目录下，然后修改 `setting.json` 中的 `desktopPet` 字段为桌宠包的文件夹名称，重启程序即可。桌宠包的标准结构应包含 `config.json` 和 `res/` 目录（内含 `stand.gif`、`drop.gif`、`icon.gif`），如有 `plugin/` 目录也一并放入。

---

**Q: 为什么我启动了程序，但是玩的并不是我粘贴进来的桌宠包？**

**A:** 检查 `setting.json` 中的 `desktopPet` 字段是否修改为你粘贴进来的桌宠包文件夹名称。程序只会加载该字段指定的桌宠包，而不是自动识别 `data/` 目录下所有的文件夹。

---

**Q: 启动程序后，桌宠的显示大小太大/小了**

**A:** 在 `setting.json` 中修改 `imageSize` 字段，设置为想要的 `[宽度, 高度]` 像素值，重启程序即可生效。

---

**Q: 怎么屏蔽日志中我不想看见的信息？**

**A:** 在 `setting.json` 中调整 `logLevel` 字段的值即可：

| 值  | 级别     | 输出内容                 |
| --- | -------- | ------------------------ |
| 0   | debug    | 输出所有日志             |
| 1   | info     | 输出 info 及以上         |
| 2   | warn     | 输出 warning 及以上      |
| 3   | error    | 只输出 error 和 critical |
| 4   | critical | 只输出 critical          |

例如设为 `3`，则 debug、info、warn 级别的日志都不会记录，日志文件会干净很多。

---

**Q: 怎么调整桌宠的物理行为？**

**A:** 在桌宠目录下的 `config.json` 中调整 `acc`、`fri`、`ela` 三个字段：

| 字段  | 说明                                                                          |
| ----- | ----------------------------------------------------------------------------- |
| `acc` | 加速度（重力），`[水平, 垂直]`，垂直值越大下落越快                            |
| `fri` | 摩擦力，`{top, bottom, left, right}`，值越大在对应边界减速越快                |
| `ela` | 弹性系数，`{top, bottom, left, right}`，值越大碰撞后反弹越强（100为完全弹性） |

修改后保存文件，重启程序即可生效。详见 `docs/中文/桌宠配置.md`。

---

**Q: 怎么加载插件？**

**A:** 在桌宠目录下的 `config.json` 中，将插件文件夹名称添加到 `plugin` 数组里即可。详见 `docs/中文/桌宠配置.md`。

---

**Q: 桌宠卡在屏幕边缘怎么回事？**

**A:** 检查 `config.json` 中的 `fri`（摩擦力）值是否大于 `acc`（加速度）。当摩擦力大于重力加速度时，桌宠碰到边缘后摩擦力会完全抵消重力，导致速度归零后无法再移动，看起来像是卡住了。适当调低 `fri` 或调高 `acc` 即可解决。

**Q: 为什么角色的素材和角色的插件行为不匹配？**

**A:** 插件行为与角色素材是两个独立的部分，插件控制的是桌宠的物理运动、弹窗交互等逻辑功能，而素材（如 `stand.gif`、`drop.gif`）只负责外观显示。两者的匹配需要插件内部主动调用 `loadMovie()` 方法来切换对应的素材文件。如果插件没有针对当前角色编写素材切换逻辑，就会出现行为与素材不匹配的情况。这通常需要插件开发者适配不同角色的素材命名和路径。

**Q: 报错：KeyError: '\_type'，是怎么回事？（跟插件的menu有关）**

**A:** 这是因为 `menu` 字典本身也需要 `_type` 字段，且必须为 `"/"`。正确的 `menu` 结构应该是：

```python
menu = {
    "_type": "/",
    "消息": {
        "_type": "$",
        "_activate": Information()
    }
}
```

如果 `menu` 顶层缺少 `"_type": "/"`，程序无法识别这是一个子菜单结构，就会报 `KeyError: '_type'`。
